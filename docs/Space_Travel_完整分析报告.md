## 三、GAIA 协议详解

### 3.1 数据包格式

#### 命令包 (手机 → 耳机)
```
字节0-1: [0xFF, 0x04] 包头
字节2-3: [类型, 序列号]
字节4:   [0x00]
字节5-6: Vendor ID (0x001D = 29)
字节7:   Feature ID
字节8+:  Command ID + Payload
```

#### 响应包 (耳机 → 手机)
```
字节0: [0x00]
字节1: Vendor ID (29)
字节2: Feature ID
字节3: Command ID (带类型位)
字节4+: Payload
```

### 3.2 Feature ID 映射

| Feature | 十进制 | 功能 | 支持的命令 |
|---------|--------|------|-----------|
| 0x00 | 0 | 设备管理 | 1(电池), 12(配置查询), 19-21(状态) |
| 0x01 | 1 | 基础功能 | 5(版本), 7(状态), 12(配置), 13(设备状态) |
| 0x03 | 3 | ANC V2 | 3(获取模式), 4(设置模式) |
| 0x07 | 7 | EQ/音乐处理 | 0(状态), 1(预设列表), 3(选择预设) |
| 0x0A | 10 | 编解码器 | 2(LDAC), 3(LHDC) |
| 0x0B | 11 | 状态查询 | 2(查询), 3(设置) |

## 五、连接失败原因分析

### 5.1 我们的 App 连不上
```
RFC_EVENT_DM ← 耳机主动拒绝
```

**原因**: 耳机验证 APP 签名，非官方签名被拒绝。

### 5.2 官方 APP 能连上
- 使用官方签名证书
- 完成初始配对认证
- 通过 SDP 查询获取正确的 RFCOMM channel

## 九、结论

Space Travel 耳机使用 **GAIA V3 协议** 通过 **RFCOMM Channel 16** 通信。连接需要官方签名验证，第三方 APP 无法直接连接。建议通过 Frida hook 官方 APP 分析完整的认证流程。
