## 关键发现

### 1. 没有显式认证握手

从抓包数据看，**没有看到显式的认证/签名验证步骤**。通信直接从功能查询开始。

### 2. 连接建立方式

```
手机 → 耳机: RFCOMM Channel 16
MTU: 990 bytes
UUID: 00001101-0000-1000-8000-00805F9B34FB (SPP)
```

### 3. 可能的验证机制

**SDP 服务发现验证**:
- 耳机在 RFCOMM 连接建立时检查 SDP 查询
- 只有特定 UUID 的 APP 才能获取正确的 RFCOMM channel

**Android 系统级验证**:
- `BluetoothDevice.createRfcommSocketToServiceRecord()` 会触发 SDP 查询
- 耳机可能只响应特定 APP 的 SDP 请求

### 4. 为什么我们的 APP 连不上

```
我们的 APP:
1. 调用 createRfcommSocketToServiceRecord(MOONDROP_UUID)
2. 系统发起 SDP 查询
3. 耳机拒绝响应 (RFC_EVENT_DM)
4. 连接失败

官方 APP:
1. 调用 createRfcommSocketToServiceRecord(MOONDROP_UUID)
2. 系统发起 SDP 查询
3. 耳机响应，返回 channel 16
4. 连接成功
```

## 结论

1. **没有应用层认证**: GAIA 协议本身没有签名验证
2. **验证在连接层**: RFCOMM 连接建立时已经完成验证
3. **可能的机制**: SDP 服务发现 + Android 蓝牙权限
