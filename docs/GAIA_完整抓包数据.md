### 三、数据包格式分析

#### V3 命令格式 (手机 → 耳机)
```
字节0-1: [0xFF, 0x04] 固定包头
字节2-3: [0x00, 0x01] 命令类型
字节4:   [0x00] 序列号
字节5-6: Vendor ID (小端) = 0x001D = 29
字节7:   Feature ID
字节8:   Command ID / Payload
```

#### V3 响应格式 (耳机 → 手机)
```
字节0:   [0x00] 固定
字节1:   Vendor ID = 29
字节2:   Feature ID
字节3:   Command ID (带类型位)
字节4+:  Payload
```

## 关键发现

1. **认证机制**: 耳机在连接后立即发送状态查询 (Cmd=7)，可能在验证 APP 身份
2. **版本协商**: 交换固件版本 "1.0.0"
3. **配置同步**: 通过 Cmd=12 交换多项配置
4. **无加密**: 数据以明文传输，没有看到加密握手
